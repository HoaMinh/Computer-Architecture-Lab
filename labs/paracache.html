DANH SÁCH BÀI THỰC HÀNH PARACACHE

Bài thực hành Vào trình

BÀI 5.1: Mô phỏng Cache ánh xạ trực tiếp (Direct Mapped)

Cấu hình: Cache Size = 64 Bytes, Memory Size = 512 Bytes, Offset Bits = 3.

Thực hiện các lệnh LOAD(4), LOAD(A), LOAD(2C). Quan sát cách địa chỉ Hex chuyển sang nhị phân và phân bổ vào Tag, Index, Offset.

Yêu cầu: Ảnh chụp kết quả sau lệnh LOAD(2C) và giải thích sự thay đổi Valid bit.

BÀI 5.2: Cache hoàn toàn phối hợp (Fully Associative)

Thực hiện tương tự Bài 5.1 nhưng chọn chế độ Fully Associative. Thử nghiệm với các chiến thuật thay thế khối: FIFO và LRU.

Yêu cầu: So sánh số lần Hit/Miss của FIFO và LRU sau 10 lệnh truy xuất ngẫu nhiên.

BÀI 5.3: Cache phối hợp theo tập hợp 2 chiều (2-Way Set)

Cấu hình Cache phối hợp theo tập 2 chiều. Quan sát cách một địa chỉ có thể được đặt vào một trong hai vị trí trong tập (Set) tương ứng.

Yêu cầu: Ảnh chụp bảng Cache và chỉ rõ vị trí các Set.

BÀI 5.4: Cache phối hợp theo tập hợp 4 chiều (4-Way Set)

Mở rộng mô phỏng lên 4-Way Set Associative. Phân tích lợi ích của việc tăng số "Way" đối với tỉ lệ Miss do tranh chấp (Conflict Miss).

Yêu cầu: Nhận xét về độ phức tạp của việc tìm kiếm Tag khi số Way tăng lên.

BÀI 5.5: Phân tích hiệu suất các loại Cache

Sử dụng chế độ Cache Type Analysis để chạy cùng một chuỗi lệnh trên nhiều loại cấu hình Cache khác nhau cùng một lúc.

Yêu cầu: Lập bảng so sánh Hit Rate của 4 loại: Direct, Fully, 2-Way, 4-Way.

BÀI 5.6: Mô phỏng Bộ nhớ ảo (Virtual Memory)

Cấu hình: Physical Pages = 128, Page Size = 4, Virtual Pages = 2048.

Sử dụng Gen. Random để tạo 10 lệnh. Quan sát quá trình ánh xạ từ Virtual Address sang Physical Address thông qua TLB và Page Table.

Yêu cầu: Giải thích hiện tượng Page Fault nếu xảy ra trong quá trình mô phỏng.

BÀI 5.7: Tính toán AMAT (Average Memory Access Time)

Dựa trên kết quả thống kê (Statistics) sau khi chạy 20 lệnh, hãy tính thời gian truy cập bộ nhớ trung bình dựa trên tỉ lệ Hit/Miss và độ trễ (Latency) thiết lập.

Yêu cầu: Trình bày công thức và kết quả tính toán AMAT cụ thể.
