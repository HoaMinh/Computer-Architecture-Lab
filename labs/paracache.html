BÀI 5.1: Mô phỏng Cache ánh xạ trực tiếp (Direct Mapped)

Cấu hình: Cache Size = 64B, Memory Size = 512B, Offset Bits = 3[cite: 3, 4, 5, 12, 13, 14].
Địa chỉ: LOAD 4H, AH, 2CH[cite: 6].

1. Chọn Direct Mapped Cache, Write Back và Write On Allocate[cite: 8, 9, 10].

2. Phân tích 9 bit địa chỉ: 3 bit TAG, 3 bit INDEX, 3 bit OFFSET[cite: 16].

3. Nhấn Next để theo dõi quá trình nạp dữ liệu từ Memory Block (64 khối) lên Cache Table (8 khối)[cite: 17, 18, 23].

Yêu cầu: Nhận xét sự thay đổi của Valid bit, Tag và dữ liệu sau lệnh LOAD(2C)[cite: 18, 24].

BÀI 5.2 - 5.4: Cache phối hợp & Chính sách thay thế

Thực hiện mô phỏng tương tự bài 5.1 với các kiểu tổ chức Cache khác nhau để so sánh hiệu quả của các chính sách thay thế FIFO và LRU[cite: 25, 26, 27].

  • Bài 5.2: Fully Associative Cache (Hoàn toàn phối hợp)[cite: 25].
  • Bài 5.3: 2-Way Set Associative Cache (Phối hợp tập hợp 2 chiều)[cite: 26].
  • Bài 5.4: 4-Way Set Associative Cache (Phối hợp tập hợp 4 chiều)[cite: 27].
Yêu cầu: Cho nhận xét về ưu điểm/nhược điểm của FIFO so với LRU trong từng cấu hình.

BÀI 5.5: So sánh hiệu năng giữa các loại Cache

Sử dụng tab Cache Type Analysis để đánh giá tỉ lệ Hit/Miss của 4 loại Cache cùng lúc[cite: 28, 30, 31].

Thiết lập: Trace file trace_mix.trc, Cache Size = 8KB, Block = 64B, LRU, Write-Through[cite: 34, 35, 36, 37, 38].
Các loại chọn: Direct-Mapped, 2-Way, 4-Way, Fully Associative[cite: 39, 40, 41, 42, 43].

Nhấn Gen 88 Random Instruction và thực hiện bằng Fast Forward để xem kết quả thống kê[cite: 45, 46, 47].

Yêu cầu: Nộp ảnh chụp kết quả và nhận xét tại sao Direct-Mapped thường có nhiều Conflict Miss hơn[cite: 51, 55, 56].

BÀI 5.6: Ảnh hưởng của kích thước khối (Block Size)

Đánh giá sự thay đổi hiệu năng khi thay đổi kích thước Block (8B, 16B, 32B) trong khi giữ nguyên tổng dung lượng Cache 64B[cite: 60, 62, 70, 74].

Cấu hình cố định: Memory 2048B, LRU, Write-Back/Write-Allocate[cite: 65, 66, 67].
Block Size (Offset) Direct (Hit%) 2-way (Hit%) 4-way (Hit%) Fully (Hit%)
8 B (Offset=3) [cite: 68]
16 B (Offset=4) [cite: 74]
32 B (Offset=5) [cite: 74]
Yêu cầu: Giải thích tại sao tăng Block size không phải lúc nào cũng cải thiện hiệu năng[cite: 75, 81].

BÀI 5.7: Mô phỏng bộ nhớ ảo (Virtual Memory)

Minh họa cách địa chỉ ảo được ánh xạ sang địa chỉ vật lý qua TLB và Page Table[cite: 84, 86, 96, 97].

Cấu hình mặc định:
- Physical Page: 128B, Offset: 2 bits (4B/Trang)[cite: 89, 90].
- Virtual Memory: 2048B (Địa chỉ 11 bits)[cite: 91, 95].
- TLB: 10 entries[cite: 92, 93, 104].

Thực hiện Gen. Random 10 lệnh LOAD đầu tiên[cite: 101, 105]. Quan sát Hit Rate (thường là 0% ở lần đầu) và giải thích hiện tượng Page Fault[cite: 109, 110].

Yêu cầu: Tiếp tục chạy thêm 10-20 lệnh và mô tả diễn biến khi TLB bắt đầu được lấp đầy dữ liệu[cite: 111, 112].
